我关闭了我的服务器

发表于 2026-05-02 分类于 杂谈

今天，2026 年 5 月 2 日，我正式关闭了自己的服务器。

从记事起，我总是对新鲜事物满怀热情。小时候尝试各种运动，学习钢琴和书法；大一些开始接触电脑，不断折腾硬件：装台式机、显卡改水冷等等；再大一些就到了大学，大一学完编程之后开始折腾 NAS 存储，买了绿联 NAS 之后又开始折腾公网访问，公网访问解决完之后开始部署个人网站，最后买阿里云的 VPS 进行备案。

过程是曲折而快乐的。买完阿里云服务器之后，开始用域名搭建各种服务，主要是影视服务器。硬盘买了一块又一块，16T+4T+3T×2。结果搭完之后发现家里上传带宽不够，需要压缩画质。NAS 处理器性能太差，压缩会卡，所以利用之前剩下的台式机配件组了台服务器，上面安装 Windows + Jellyfin 进行影视解码。

在这暑假期间，网站也是更新了很多次，从一开始的 WordPress 框架，换到 Halo 框架。最后不满意效果，又换回了 WordPress。域名也买了三个，两个 .top 后缀的，一个 .com 后缀的，备案了两个。

再之后开学了，突然发现学校的网络带宽很大，上下行千兆对等，于是购入一台零刻的飞牛 NAS（全固态）放在宿舍，网站也从 VPS 上迁移到了 NAS 上。之后想着把影视库也迁移过来，于是低价购入了极空间 Z2 Pro，让家里把 16T 的硬盘寄过来。之后还买了 115 网盘的 40T 空间 8 年会员，做了 302 直链播放。

全部搞定之后索然无味，自己也没怎么看。网站也是偶尔更新。与此同时，家里的服务器被海外黑客入侵了，所以我重装了系统，改成了 Ubuntu。之后尝试在上面部署 MC 服务器，也运行了一段时间，但我自己不怎么玩，所以后续也没有结果。

之后到了大二的寒假，我重新把几台设备和服务器整理了一下，重新美化了个人网站。

2025 年和 2026 年是 AI 急速爆发的两年。在这个寒假，我几乎用遍了所有的 AI 模型，在自己的服务器上部署了一个又一个开源项目。尝试了 Vibe Coding，用了很多 Agent 框架，最有用的是 Dify，可以可视化搭建工作流。一个假期 API 都烧了 500+ RMB，在 GitHub 上上传了几个用处不大的项目。

之后就到了这个学期，由于我报了数学双学位，课业非常繁重，之前部署的所有服务已经成为了我的负担。所以我开始不断缩减和优化自己的工具使用。AI 不再调用 API 使用，也不使用 Workflow，而是老老实实开了 Kimi 的会员。编程上也不接自己的 API 了，直接开 Cursor 的年度会员。学校的台式机太占位置，而且我也不再打游戏，于是在 4 月初校庆期间以略低的价格本校出掉，恰逢内存条飞涨，几乎覆盖了成本。之后电脑换成了现在的 ThinkBook 16+ 锐龙版，所有的主要工作都放在上面，省事且好整理，MacBook 只用来写代码或者需要长续航的场景下用。我的三台 NAS 也不再折腾，配置好了其中两台自动备份到第三台之后，就只做数据备份的用途了。影视库还在，但我几乎不看。

快五一的时候，小米的 MIMO 模型发布了，一起来的还有一个百万亿 Tokens 发放计划。我凭借之前 API 的消费账单，以及 GitHub 上几个小小的项目，领到了一个月的标准 Token Plan 套餐。刚好之前的博客我觉得很麻烦，因为是动态站点，必须去后台才能写新的文章。而且星度主题有些华而不实，不如静态博客来得简约。此外我在网站上也想收缩一下，专注于内容更新，所以想着换成 Hexo 静态博客，之后的服务器什么也不去续费了，没有意义。刚好 GitHub 上有一个开源项目 Qexo，可以管理 Hexo 博客发布的全过程，但是部署很麻烦，就想着重构成 Electron 软件。那就刚好，OpenCode 接入小米的 Token Plan，开始重构。折腾了差不多两天，最终完成，页面长这样：

刚开学的时候试着自己部署一个 Overleaf，后来兜兜转转找到了一个本校研究生学长的开源项目。他把 Overleaf 社区版拿去改造，加了很多 Pro 版才有的功能。我们学校的 Overleaf 用的就是他的镜像。部署很困难，各种环境不兼容，但最后还是成功了。此外我还搓了一个自动备份的工具，可以每天自动拉取我平台上的文件进行备份：

回想过去，确实在服务器、网络这一块折腾到头了，有些身心俱疲。看一眼自己要处理的域名就头大：

是动态站点，后面不续服务器了考虑转成静态站点。

坦率地说，折腾了这么多东西也不是毫无用处，至少学到了一些技能，对软件和 Web 开发也有了一定的了解。最重要的是对 AI 的使用已经到了炉火纯青的地步，至少在调用方面胜过了身边所有人，可以认为是全国用的最舒服的那一批，彻底融入了生活又不会过分依赖，自己也能写一些工作流和交互的调用。举个例子，这篇文章并不是最初的版本，我用 ai 润色过。但如果我不说，大概也没人能发现。

但现在，事情越来越多，也该放掉一些东西。之前的 NAS 和台式机已经处理完，现在轮到服务器了。今天把静态博客正式上线（ https://www.chrono-well.top/ ），把之前自己搭建的 LaTeX 平台的文件备份出来，正式关闭线下物理机。至于 NAS 和云端的一些服务，挺感激自己当初部署时的那份认真，当时是朝着永不维护、一直运行下去的目标构建的，所以暂时不用管理，等放暑假了再说，到时候连着 VPS 和备案一起处理掉。

高中的时候学诗词，总是觉得很枯燥乏味。但到了现在，却有了不一样的感受。离骚里的日月忽其不淹兮，春与秋其代序，以及陶渊明写的善万物之得时，感吾生之行休，时常萦绕在我的脑海里。

这学期在思想上很幸运，一是遇到了一个非常好的马原老师，刚刚北大文学博士毕业。他的课程对我整个思想产生了极大的冲击，几乎重塑了我的价值观，以及与人相处的方式和原则；二是选了一门诗词的通识课，感受到了一些不一样的东西。此外还有很多很幸运的事，有时候真的觉得，命运是存在的，世界有自己的定数。

回归到一开始的话题，今天我的线下物理机就正式关停了。心里难免有些伤感，但又不得不做。又想起一句诗：流水今日，明月前身。一件事放在时间轴上看，不过也就是一个点，人还是得往前走。现在的大环境不好，大家的压力都很大，但这个过程中人和环境、社会产生更多的交互，也就有了更多的实践，这也是马克思认为的真正的人，也是人和 AI 最本质的区别。
